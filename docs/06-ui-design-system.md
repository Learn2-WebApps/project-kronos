# Project KRONOS UI 디자인 시스템

> 본 문서는 Project KRONOS의 시각적 정체성, 사용자 인터페이스(UI) 구성 요소 및 사용자 경험(UX) 표준을 정의합니다.

---

## 1. 디자인 철학

**"수사 기록물" 컨셉의 시네마틱 비주얼 노벨 + 메신저 하이브리드.**
*   **디지털 다크 톤(검정 배경)** × **아날로그 종이 질감(베이지 기록물)**의 대비
*   캐릭터 일러스트 중심의 시네마틱 컷 배치
*   모든 "기록"은 종이 질감(찢어진 가장자리, 그레인) 사용
*   모든 "대화·실시간 UI"는 다크 메신저 톤 사용
*   미니멀하지만 영화적 디테일(비네팅, 그레인, 광원) 적극 활용

*   **Body**: 15px / 1.5 / 400
*   **Bubble**: 14px / 1.5 / 400
*   **Caption**: 13px / 1.4 / 400

## 4. 레이아웃

### 4-1. 메인 인터뷰 화면 (v2.2 업데이트)

**전체 비율: 상단 일러스트 70% + 하단 채팅 30%** (또는 연출에 따라 50:50 가변)

```text
┌─────────────────────────────────────────────────┐
│ [상단 바 56px] 로고 | 타이머 27:43 | [📋 인벤토리] │
├─────────────────────────────────────────────────┤
│                                                 │
│           [캐릭터 일러스트 영역 70%]            │
│           - 상반신 PNG 4종 오버레이                │
│           - 배경: 단순 다크 그라데이션 유지         │
│           - AI 응답 <emotion>에 따른 표정 표시     │
│           - 표정 전환 시 200ms fade transition    │
│           - 우상단: 남은 턴 인디케이터             │
│                                                 │
├─────────────────────────────────────────────────┤
│ [채팅 영역 30%]                                 │
│   ◯ 정민호: "..." (말풍선)                      │
│   "..." :사용자 (말풍선)                        │
│   [입력창 ────────────────] [전송]   [🔍ECHO] │
└─────────────────────────────────────────────────┘
```

*   **상단 바 (Top Bar)**: 56px. 우측에 단서 카운터 대신 **인벤토리 아이콘** 배치.
    *   인벤토리 아이콘: 클립보드 SVG(24px). 새 단서 발견 시 우상단에 4px 펄스 점 표시.
*   **일러스트 영역 (Illustration Area)**: 전체 높이의 70% (min 480px).
    *   **배경**: 단순 다크 그라데이션 유지 (v2.1과 동일).
    *   **캐릭터 표시**: 상반신 PNG 4종({character}-normal, -thinking, -surprise, -worry) 오버레이.
    *   **표정 제어**: AI 응답의 `<emotion>` 태그 값(`normal`, `thinking`, `surprise`, `worry`)에 따라 해당하는 PNG 표시.
    *   **전환 효과**: 표정 전환 시 200ms fade transition 적용.
    *   **턴 인디케이터**: 우상단(top 24px, right 32px). 캐릭터 컬러 2px 좌측 라인 + "남은 턴" 라벨 + "7 / 10" 숫자.
*   **채팅 영역 (Chat Area)**: 전체 높이의 30% (min 240px). 채팅 내부 max-width 880px.
*   **ECHO 마스코트**: 우하단 고정, 80×80px, 채팅 입력창 우측 상단 상주.

### 4-2. 단서 진도율 표시 정책

*   **몰입 보호**: 게임 플레이 중에는 전체 단서 개수(41개)를 의도적으로 숨깁니다.
*   **인벤토리**: 플레이어는 인벤토리 아이콘을 클릭하여 지금까지 수집한 단서 목록만 확인할 수 있습니다.
*   **공개 시점**: 진도율(%)과 전체 개수는 게임 종료 후 평가 화면(`docs/05-evaluation-rubric.md`)에서만 공개됩니다.
*   **피드백**: 새 단서 발견 시 우상단 단서 토스트와 인벤토리 아이콘의 펄스 점으로 획득 사실을 알립니다.

### 4-3. 캐릭터 선택 화면 (v2.2 신규)

오프닝 카드와 메인 인터뷰 사이에 위치. 5명의 용의자를 카드 형태로 나열.

*   **배경**: `var(--bg-base)` 다크 톤 + 그레인 오버레이.
*   **상단 헤더 (56px)**: "용의자 선택" 타이틀, 좌측 뒤로가기, 우측 남은 시간.
*   **메인 영역**: 5개 카드 가로 배열 (반응형, 1920px에서 가로 1열).
*   **각 카드**: `{character}-background.png` 풀블리드 표시, 하단 그라데이션 오버레이 위 캐릭터 이름·직책 표시.
*   **카드 크기**: 360×600px, gap 24px, hover 시 lift 8px + 캐릭터 색상 글로우.
*   **인터뷰 진행 상태 표시**: 카드 우상단에 남은 턴 수 (예: "7/10"), 0턴 시 비활성화 처리.
*   **동작**: 클릭 시 해당 캐릭터의 메인 인터뷰로 이동, 인터뷰 종료 후 이 화면으로 복귀.

## 7. 캐릭터 에셋 사양 (v2.2 개정)

### 7.1 파일 구조

각 캐릭터당 6개 파일, ECHO는 1개 파일. 총 31장.

| 파일명 | 용도 | 해상도 | 사용 화면 |
|--------|------|--------|----------|
| {character}-normal.png | 상반신, 평온 | 1200×1600 PNG 투명 | 메인 인터뷰 (기본 상태) |
| {character}-thinking.png | 상반신, 생각/회피 | 1200×1600 PNG 투명 | 메인 인터뷰 (회피·고민 시) |
| {character}-surprise.png | 상반신, 놀람 | 1200×1600 PNG 투명 | 메인 인터뷰 (모순 지적·자백 시) |
| {character}-worry.png | 상반신, 우려/긴장 | 1200×1600 PNG 투명 | 메인 인터뷰 (압박 받을 때) |
| {character}.png | 전신 일러스트 | 1200×1800 PNG 투명 | 오프닝 카드 5인 소개, 추리 노트 프로필 |
| {character}-background.png | 캐릭터+배경 합본 | 1920×1080 JPG/PNG | 캐릭터 선택 화면 |
| echo.png | ECHO 마스코트 | 400×400 PNG 투명 | 메인 화면 우하단 고정 |

### 7.2 캐릭터 ID 매핑

- jung-minho (정민호)
- han-jihun (한지훈)
- yoon-seokyung (윤서경)
- kang-hyerin (강혜린)
- oh-sera (오세라)

### 7.3 표정 시스템

AI 응답의 <emotion> 태그는 4가지 값만 사용:
- **normal**: 평온, 기본 상태
- **thinking**: 생각, 회피, 시선 흐림
- **surprise**: 놀람, 모순에 찔림, 자백 직전
- **worry**: 우려, 긴장, 압박 받음

표정 전환 방식: AI가 대화 맥락에 맞춰 자연스럽게 선택. 강제 트리거 규칙 없음. 캐릭터 프롬프트에 "상황 가이드" 방식으로 제시.

### 7.4 파일 경로

- `/public/characters/{character-id}/{filename}.png`
  - 예: `/public/characters/jung-minho/jung-minho-normal.png`
- ECHO: `/public/echo.png`

## 8. ECHO 마스코트 (우하단)

*   **위치**: `position: fixed`, 우하단 24px 마진
*   **크기**: 80×80px 원형 (echo.png 사용)
*   **보더**: `2px solid var(--accent-amber)`, 외부 glow `box-shadow: 0 0 16px rgba(212,165,116,0.3)`
*   **배경**: `var(--bg-elevated)`
*   **인터랙션**: 클릭 시 위쪽으로 말풍선 패널이 슬라이드 업
*   **말풍선 패널 사양**:
    *   **위치**: 마스코트 위쪽 16px 띄움, 우측 정렬
    *   **크기**: 360×auto, max-height 480px
    *   **배경**: `var(--bg-elevated)`, `1px solid var(--accent-amber)`
    *   **꼬리**: 우하단 → 마스코트 방향
    *   **내용**: 기능 버튼 5개(정보 요약 / 시간선 / 진술 비교 / 추천 질문 / 힌트) + 자유 입력창
    *   **닫기**: 우상단 X 버튼

## 9. 종이 기록물 컴포넌트

### 9-1. 공통 사양

**적용 대상:**
*   오프닝 카드(사건 브리핑)
*   추리 노트(인벤토리)
*   단서 카드 / 단서 토스트
*   평가 보고서
*   디브리핑 화면

**사양:**
*   **배경**: `var(--paper-base)` + `paper-texture.png`(그레인·얼룩) 오버레이
*   **가장자리**: SVG `torn-edge mask` (상하 또는 사방, 8~12px 불규칙)
*   **그림자**: `box-shadow 0 12px 32px rgba(0,0,0,0.5), 0 0 0 1px var(--paper-shadow)`
*   **텍스트**: `var(--paper-ink)`, `Noto Serif KR` 또는 `Gaegu`
*   **장식**: 좌상단 황동 핀(SVG, 24px), 또는 가죽 끈/스티커
*   **키워드 강조**: 형광펜 효과 (`background: rgba(212,165,116,0.4), padding 0 2px`)
*   **손글씨 메모**: `Gaegu` 폰트, 약간 회전(`transform: rotate(-1.5deg)`)

### 9-2. 오프닝 카드 (v2.2 업데이트)

*   **5인 미니카드**: `{character}.png` (전신 일러스트) 사용.
*   **배치**: 전신 일러스트를 카드 하단에 정렬.
*   **카드 크기**: 기존 유지.

### 9-3. 추리 노트 (v2.2 업데이트)

*   **캐릭터 프로필 영역**: 단서 카드 클릭 시 우측 상세 영역 상단에 해당 캐릭터의 `{character}.png` 전신 일러스트 (200×300px) 표시.
*   **배치**: 캐릭터 이름·직책 옆에 배치.

### 9-4. 단서 토스트 (소형 종이)

*   320×88px, 우상단 슬라이드 인
*   "📎 새 단서 발견" 라벨(미니 황동 클립 SVG) + 단서 코드 + 짧은 설명
*   3초 후 페이드 아웃

**용도**: 메인 화면 진입 시 또는 햄버거 메뉴 클릭 시 표시.
*   **폭**: 360px 사이드 시트
*   **배경**: `var(--bg-elevated)`
*   **항목 높이**: 80px
*   **항목 구성**: 좌측 56px 원형 캐릭터 아이콘 + 이름 + 역할(`--text-muted`) + 우측 "마지막 발언 미리보기" + 턴 카운터(7/10)
*   **활성 상태**: 좌측 3px 캐릭터 컬러 라인 + 배경 살짝 밝게
*   **조사 완료**: `opacity: 0.5`, "조사 완료" 회색 태그

## 11. 애니메이션

*   **페이지 페이드 인**: 300ms `ease-out`
*   **캐릭터 표정 교체**: `cross-fade 200ms`
*   **말풍선 등장**: `translateY(8px)` → 0, `opacity 0 → 1`, 200ms
*   **단서 토스트**: `slide-in 320ms cubic-bezier(0.2, 0.8, 0.2, 1)`, 페이드 아웃 400ms
*   **ECHO 말풍선**: `scale(0.95)` → 1, `opacity 0 → 1`, 220ms
*   **종이 카드 등장**: 살짝 회전 + 페이드, 360ms

## 12. AI 응답 태그 시스템

캐릭터 AI 응답에 다음 태그 포함:
*   `<emotion>normal|thinking|surprise|worry</emotion>` → 일러스트 표정 전환
*   `<clues>F-12, L-01</clues>` → 진도율 카운트
*   `<scene>office|meeting-room|lounge</scene>` (선택) → 배경 전환

## 13. 금지 디자인 항목
