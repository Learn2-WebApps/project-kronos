# 게임 진행 플로우 (Game Progression Flow)

> 본 문서는 학습자가 게임을 시작하여 종료하기까지의 전체 사용자 경험(UX)과 시스템 동작 흐름을 정의합니다.

---

## 1. 전체 게임 구조 개요

### 1.1 게임 단계
1. **오프닝 (Opening)**: 사건 브리핑 및 역할 부여
2. **메인 조사 (Main Investigation)**: 용의자 인터뷰 및 ECHO 보조 활동
3. **최종 제출 (Final Submission)**: 범인 지목 및 추리 근거 작성
4. **평가 화면 (Evaluation)**: 조사 진도율 및 추리 정확도 보고서 확인
5. **디브리핑 (Debriefing)**: 자기 성찰 질문 및 핵심 학습 정리 (선택)

### 1.2 예상 소요 시간
- **전체**: 30~40분
- **오프닝**: 2~3분
- **메인 조사**: 25~30분
- **최종 제출**: 3~5분
- **평가 화면**: 자율

---

## 2. 단계 1: 오프닝

### 2.1 ECHO 사건 브리핑 카드
*   **형식**: 카드 UI (증거물 또는 문서 형태의 연출)
*   **내용**:
    *   **사건 제목**: "NovaTech 한지훈 상무 부당해고 의혹 사건"
    *   **배경 요약**: 익명 제보 메일, 위조된 영수증 12장, 갑작스러운 해임 통보, 변호사 측의 항의.
    *   **학습자 역할**: 본사 윤리감사관 (Internal Auditor)
    *   **등장인물 5명 소개**: 이름, 직책, 사건과의 관계 요약 (예: 정민호 - 한지훈의 최측근 팀장)
    *   **최종 목표**: "진범을 특정하고, 그 판단을 뒷받침할 결정적 단서를 확보하시오."
*   **인터랙션**: "조사 시작" 버튼 클릭으로 메인 조사 단계 진입.

### 2.2 게임 규칙 안내
브리핑 직후 하단 또는 팝업으로 짧은 가이드 제공:
*   **턴 제한**: 캐릭터당 최대 10턴 (질문 10회) 가능.
*   **시간 제한**: 전체 조사 시간 30~40분 부여.
*   **ECHO 지원**: 우측 패널에서 언제든 호출 가능 (턴 소모 없음).
*   **강제 종료**: 시간 또는 전체 턴 소진 시 자동으로 제출 단계 진입.

## 3. 단계 2: 메인 조사 화면

### 3.1 화면 레이아웃
*   **좌측 사이드바 (Navigator)**: 용의자 5명 목록 (프로필 이미지 + 이름 + 남은 턴 수 표시).
*   **중앙 영역 (Chat View)**: 선택된 캐릭터와의 인터뷰 창. 캐릭터별 전용 배경 일러스트 적용.
*   **우측 사이드바 (ECHO Panel)**: 접힘/펼침 가능한 보조 패널.
*   **상단 바 (Status Bar)**: 전체 남은 시간, 총 단서 발견 현황 (예: 📂 발견된 단서 12/41).

### 3.2 캐릭터 선택 및 대화 흐름
*   용의자 아이콘 클릭 시 해당 캐릭터와의 대화로 즉시 전환. 기존 대화 기록은 유지됨.
*   캐릭터 첫 진입 시 자동 발화 (예: 정민호 - "감사관님, 무슨 일로 부르셨습니까? 제가 도울 일이 있다면 말씀하시죠.")

### 3.3 턴 및 시간 관리
*   사용자 질문 1회 전송 시 해당 캐릭터의 턴이 1씩 감소.
*   10턴을 모두 소진한 캐릭터는 목록에서 비활성화되며, "더 이상 질문할 수 없습니다" 메시지 표시.
*   전체 시간이 5분 남았을 때 상단 바에 경고 표시.

### 3.4 단서 자동 수집 로직
*   AI 응답 내 `<clues>ID</clues>` 태그를 실시간 파싱하여 '획득 단서 목록'에 추가.
*   **백업 매칭**: 태그가 누락되어도 응답 텍스트에 '감지 트리거' 키워드가 2개 이상 포함되면 자동 획득 판정.
*   **토스트 알림**: 새 단서 획득 시 "📂 새 단서 발견: F-12 (식사 종료 시각)" 알림 출력.

### 3.5 ECHO 패널 동작
*   학습자가 명시적으로 호출(버튼 클릭 또는 질문)할 때만 동작.
*   **주요 기능 버튼**:
    *   `정보 요약`: 지금까지 얻은 진술을 표 형태로 정리.
    *   `시간선 정리`: 사건 당일의 타임라인 업데이트.
    *   `진술 비교`: 두 명의 용의자 간 진술 모순점 비교 표 출력.
    *   `추천 질문`: 현재 상황에서 조사할 가치가 있는 방향 제안.
    *   `3단계 힌트`: "질문 방향" -> "핵심 키워드" -> "직설적 힌트" 순으로 제공.

## 4. 단계 3: 최종 제출

### 4.1 제출 화면 구성
조사 시간이 종료되거나 학습자가 "최종 제출"을 클릭하면 진입합니다.
1.  **범인 지목 (Suspect)**: 5명의 인물 중 1명을 드롭다운으로 선택.
2.  **근거 단서 선택 (Evidence)**: 지금까지 발견한 단서 목록에서 최대 5개의 체크박스 선택.
3.  **추리 서술 (Reasoning)**: "왜 이 사람이 범인이며, 선택한 단서들이 어떻게 연결되는지"에 대해 200~500자 내외로 서술.

### 4.2 최종 확인
*   "한번 제출하면 수정할 수 없습니다. 정말 제출하시겠습니까?" 컨펌 창 표시.
*   남은 시간이 있다면 "조사로 돌아가기" 옵션 제공.

---

## 5. 단계 4: 평가 화면

### 5.1 처리 및 로딩
*   제출 즉시 **서술 채점 AI**를 호출하여 학습자의 추리 텍스트 분석 (약 5~10초).

### 5.2 결과 보고서 표시
`docs/05-evaluation-rubric.md` 기준에 따라 다음 항목 출력:
*   **종합 등급**: S / A / B / C / D
*   **조사 진도율**: 전체 퍼센트 및 카테고리별 도넛 차트.
*   **캐릭터별 충실도**: 캐릭터별 발견 단서 개수 및 미발견 단서 힌트.
*   **추리 정확도**: 범인 정답 여부, 증거 적합도, AI 심사평(논리성 등 4개 항목).

### 5.3 사후 활동
*   **사건 전모 공개**: 정민호-오세라 회동(F-23)을 포함한 진실 시나리오 공개.
*   **대화 복기**: 캐릭터별 전체 대화 기록을 스크롤하며 다시 보기.
*   **재도전**: 등급을 높이기 위해 다시 시작하기.

---

## 6. 단계 5: 디브리핑 (선택)

학습 전이를 위한 메타인지 질문 제공:
*   "조사 과정에서 가장 결정적이라고 느꼈던 단서는 무엇인가요?"
*   "윤서경 상무를 의심했다면, 어떤 단서 때문에 확증 편향이 생겼었나요?"
*   "ECHO의 보조 기능 중 어떤 것이 가장 도움이 되었나요?"
*   *응답은 저장되나 별도로 채점하지 않음.*

---

## 7. 게임 종료 조건 요약

다음 세 가지 상황 중 하나가 발생하면 강제로 단계 3(최종 제출)으로 이동합니다.
1.  **전체 시간 만료**: 30~40분이 지난 경우.
2.  **전체 턴 소진**: 모든 용의자 5명에게 각각 10회씩의 질문을 모두 마친 경우 (총 50턴).
3.  **수동 제출**: 학습자가 스스로 조사를 완료하고 제출 버튼을 누른 경우.

---

## 8. 상태 관리 데이터 구조 (개요)

시스템은 실시간으로 다음 데이터를 유지해야 합니다.
*   `game_clock`: 시작 시간 및 현재 경과 시간.
*   `character_turns`: 캐릭터 ID별 잔여 턴 수 (Map).
*   `dialogue_history`: 캐릭터별 질문/답변 객체 배열.
*   `collected_clues`: 발견된 단서 ID Set.
*   `echo_context`: ECHO와 나눈 대화 기록 및 생성된 표 데이터.
*   `final_report`: 학습자가 입력한 범인, 증거 리스트, 서술 텍스트.
*   `evaluation_result`: AI 채점 점수 및 등급.
